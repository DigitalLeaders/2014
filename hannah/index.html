Week 3 - CSS colors and fonts

This week we are going to learn about changing colors and fonts with CSS.

NOTE: In CSS and other programming languages, colour is spelt the American way, without the 'u'. So it's always 'color' not 'colour'!!!

How CSS Works

We are going to put our CSS into <style></style> tags that live inside of our <head></head> section.

Each block of CSS is called a "rule". Each rule is made up of a "selector" which says what bit of HTML the rule should apply to, and a set of "declarations" which say what should happen. Declarations are made up of pairs of "attributes" and "values" just like in HTML.

Here is an example CSS rule:

body {
    background: #ff0000;
}

Here's what each of the parts means:

  • body - this is the "selector". It is the HTML tag that we want the rule to apply to. In this case, body is the whole document that we can see in our browser. This could be any tag like 'h1' or 'a'.
  • { } - this marks our set of "declarations". You must always wrap your declarations in curly braces.
  • background: #ff0000; - this is a single declaration, it is made up of an attribute and a value:
    • background - this is our attribute, this is the thing that we want to change. Background means the space behind everything.
    • : - this marks the space between our attribute and our value
    • #ff0000 - this is called a 'HEX value' and represents a color.
    • ; - this marks the end of our value.

So CSS is made up of "attributes" and "values" just like in HTML, but in HTML we write them like attribute="value" and in CSS we write them like attribute: value;. This is called "syntax", it is the rules of the language, like grammar rules in English. Once you've learnt the syntax, you can find out all about what sorts of attributes and values there are by reading the MDN documentation in my useful links section.

Week 2 - Images

This week we are going to learn lots more about HTML.

Challenge 1

Can you add an image to your page?

Images are added with the <img /> tag. The image tag doesn't have a closing tag. We use an attribute called "src" to tell the tag which image to show: <img src="../shared/images/kitten.jpeg" />.

NOTE: If you're inside a folder, and want to link to an image that is in a folder above (e.g. if you are in your folder, and want to link to a shared image) you need to start the path to your image with ../

Week 1 - Learning HTML

Last week we learned about HTML tags which look like: <html></html>. HTML tags usually have an opening and closing tag.

They often also have "attributes" and "values" which provide more information about what a tag should do. Attributes and values always go together in pairs, and look like this: attribute="value". The equals sign and the quote marks are very important.

We saw a good example of this when we learned how to make links: <a href="http://www.google.co.uk">Click Me</a>. The link tag is called "a" for anchor. Inside the "a" tag we have an attribute called "href" which we can use to tell the link where it should go when it is clicked.
